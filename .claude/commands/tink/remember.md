# /tink:remember

Remember a stable preference, repeated mistake, or reusable lesson.

## Use when
- The user explicitly asks Tink to remember something.
- The same mistake appears a second time.
- A project rule clearly prevents future errors.

## Files
- `.tiny/memory/preferences.md` for stable user or project preferences.
- `.tiny/memory/mistakes.md` for repeated mistakes and prevention rules.
- `.tiny/memory/lessons.md` for generalized lessons that improve harnesses.

## Rules
- Ask before saving.
- Write short declarative bullets.
- Do not store raw logs, full diffs, secrets, one-off task progress, stale IDs, or private data.
