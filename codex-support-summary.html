Codex 지원 구현 완료

Tink Harness 변경 요약

Claude Code 기존 동작은 유지하면서, Codex에서는 $tink skill로 사용할 수 있게 확장했습니다.

핵심 변경

  • --agent=claude|codex|all 옵션을 설치기에 추가했습니다.
  • 기본값은 기존 호환성을 위해 claude로 유지했습니다.
  • Codex 설치는 .claude/를 만들지 않고 Codex skill만 설치합니다.
  • 공용 실행 상태와 하네스는 기존처럼 .tink/를 사용합니다.

Codex 사용법

  • 설치: npx tink-harness@latest install --agent=codex
  • 업데이트: npx tink-harness@latest update --agent=codex
  • 시작: $tink cast <task>
  • 지원 action: setup, cast, list, frog, weave, update

변경 파일

bin/install.jsagent 분기, Codex 설치 경로, 요약 문구 추가
templates/codex/skills/tink/SKILL.mdCodex 전용 $tink skill 신규 추가
README.md, README.ko.mdCodex 설치/업데이트/사용법 문서화
package.json설명과 keyword에 Codex 반영
tests/test_templates.pyCodex 설치, 패키징, 회귀 테스트 추가

설치 동작

Claude Code

  • .claude/commands/tink/*.md 설치
  • .claude/skills/tink/SKILL.md 설치
  • 선택 시 UserPromptSubmit hook 등록

Codex

  • $CODEX_HOME/skills/tink/SKILL.md 설치
  • CODEX_HOME 미설정 시 ~/.codex 사용
  • hook이나 Claude command는 설치하지 않음

검증 결과

  • 통과 npm test - 24 tests OK
  • 통과 node bin/install.js install --agent=codex --lang=ko --yes --dry-run
  • 테스트의 subprocess 캡처 인코딩을 utf-8로 고정해 Windows cp949 로그 예외를 제거했습니다.

주의 사항

  • ROADMAP.md는 기존 untracked 파일이라 건드리지 않았습니다.
  • 이번 변경은 npx + skill 방식의 Codex 지원이며, Codex plugin/marketplace 지원은 포함하지 않았습니다.
