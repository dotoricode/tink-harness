# v1.7.0 마이너 릴리스 이력 초안

## 문제

- surface 선택 단계에서 multi-checkbox toggle을 사용하다 보니, 커서가 이미 선택된 항목(Claude Code)에 있을 때 선택 여부를 구분하기 어려웠음.
- 컴포넌트 선택 프롬프트 메시지가 항상 동일한 고정 문자열이어서, Claude Code를 선택했을 때와 Codex를 선택했을 때의 옵션 목록이 바뀌었는지 사용자가 알기 어려웠음.

## 해결

- surface 선택을 multi-checkbox에서 단일 선택 프롬프트(Claude Code / Codex / 둘 다)로 변경했음. 선택 상태가 커서 위치와 분리되어 즉시 명확하게 보임.
- 컴포넌트 선택 메시지를 surface에 따라 다르게 출력하도록 변경했음. Claude Code는 "Claude Code 설치 항목을 선택하세요", Codex는 "Codex 설치 항목을 선택하세요"로 구분됨.

## 검증

- `npm test`로 installer 문구, 새 surface 선택 메시지, package metadata 회귀 테스트를 확인함.
- `git diff --check`로 공백 문제 없음을 확인함.
- `claude plugin validate`로 plugin 메타데이터 유효성을 확인함.
- `npm pack --dry-run --json`으로 패키지 포함 파일을 확인함.

## 참고

- minor 릴리스이므로 GitHub Release를 새로 만들고 최신 마이너 릴리스 노트를 v1.7.0으로 업데이트함.
- 이 변경은 기존 CLI flag(`--agent` 없음 정책 포함)와 non-interactive 흐름에 영향을 주지 않음.
- surface `'all'` 값은 기존 내부 코드와 동일하게 유지됨(`value: 'both'` 미사용).
